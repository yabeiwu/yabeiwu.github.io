DFTP Calculated Phonon in VASP.6.2.1

  • PUBLICATION
  • EXPERIENCES
  • DFPT-PHONON-IN-VASP6.2.1

Question:

Based on DFPT method, the phonon results by using VASP.5.4.4 and VASP.6.2.1 are different, see as following:


After carefully check, we find that the phonon dispersion are the same, but the phonon frequency are not. It seems that they have some unit difference.

And the unit seems to be the frequency unit (f.u.) problem, in the DFPT of VASP.6.2.1, it already uses the THz unit.

When we use phonopy to process the force data, it use the convert the unit to THz as default. That is to say, it will multiply by 15.6330214 again in VASP.6.2.1, this is why the results in VASP.6.2.1 has significant larger frequencies.

In order to get the right phonon frequency, you can basically use the phononpy command to avoid double counting the factor

This following figure are the new phonon results, and now they are consistent very well.

One should also note that only the DFPT method is affected in the VASP.6.2.1, if you are using finite difference method to calculate the phonon, no worries.

Question 2:

However if you have checked the FORCE_CONSTANTS carefully, you will find that the data in VASP.6.2.1 is in the new unit. So if you want to do the thing in a totally right/proper way, you should change the FORCE_CONSTANTS files, especially when you are doing some research work on, for example, the LO-TO splitting, high-order phonon interactions by using ShengBTE.

The solution is that you should write a code to divide the FORCE_CONSTANTS by 15.6330214**2，and then you will obtain a new FORCE_CONSTANTS file, and then you can use this new FORCE_CONSTANTS file to do anything you want。
